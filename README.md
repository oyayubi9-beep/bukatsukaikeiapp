# 部活会計入力アプリ セットアップ手順

このアプリは、ブラウザだけで動くPWA（サーバー不要）です。Googleスプレッドシートに直接読み書きします。動かすまでに**一度だけ**、Google Cloud側の設定が必要です。ここが最初の壁ですが、一度終えれば以降は触る必要がありません。

## できること

- 一般会計への仕訳入力（収入・支出・振替、領収書撮影つき）
- 部費入金記録（部員×月の○×－）
- 上位大会特別会計への入力（複数大会から選択）

アプリは**入力専用**です。修正・削除はスプレッドシート側で行ってください（仕様書の方針通り）。

---

## 1. Googleスプレッドシートを準備する

1. お手元の `部活会計システム.xlsx` をGoogleドライブにアップロードし、「Googleスプレッドシートで開く」で変換する（またはGoogleスプレッドシートにインポート）。
2. シート名・列の並びは元のExcelと同じなので、変換後にズレがないか一度目で確認してください。
3. スプレッドシートのURLを見て、IDをメモする。

   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```

4. 領収書を保存するためのGoogleドライブフォルダを1つ新規作成し、そのフォルダIDもメモする。

   ```
   https://drive.google.com/drive/folders/【ここがID】
   ```

5. このスプレッドシートとドライブフォルダを、会計担当者全員のGoogleアカウントに「編集者」として共有しておく。

---

## 2. Google Cloudの設定（最初の壁）

1. https://console.cloud.google.com/ にアクセスし、Googleアカウントでログイン。
2. 新しいプロジェクトを作成する（名前は「部活会計アプリ」など何でもよい）。
3. 左メニュー「APIとサービス」→「ライブラリ」から、次の2つを検索して有効化する。
   - **Google Sheets API**
   - **Google Drive API**
4. 左メニュー「APIとサービス」→「OAuth同意画面」を設定する。
   - User Type は「外部」を選択（社内向けGoogle Workspaceがあれば「内部」でもよい）。
   - アプリ名・サポートメール等を入力。
   - スコープの追加は不要（あとでアプリ側から要求します）。
   - テストユーザーに、使う予定の会計担当者のGoogleアカウントを全員追加する（公開審査に出さない限り、ここに登録した人しかログインできません）。
5. 左メニュー「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」。
   - アプリケーションの種類は「**ウェブアプリケーション**」を選択。
   - 「承認済みのJavaScript生成元」に、アプリを公開するURLを追加する（手順3で決めるURL。例：`https://your-username.github.io`）。
   - 作成すると「クライアントID」が表示されるので、コピーしてメモする（`〜.apps.googleusercontent.com` という文字列）。

---

## 3. アプリを公開する

このフォルダの中身（`index.html` 一式）を、どこかのウェブサーバーに置く必要があります。費用がかからず簡単な方法はGitHub Pagesです。

1. GitHubでリポジトリを新規作成し、このフォルダの中身をすべてアップロードする。
2. リポジトリの「Settings」→「Pages」で、公開ブランチを設定する。
3. 数分後、`https://あなたのユーザー名.github.io/リポジトリ名/` のようなURLでアプリが開けるようになる。
4. このURLを、手順2の「承認済みのJavaScript生成元」に**正確に**追加し直す（末尾のスラッシュの有無に注意。両方追加しておくと安全）。

> GitHubが難しい場合は、Claude Codeに「このフォルダをNetlifyかVercelで公開して」と頼む方法もあります。やり方は変わりますが、最終的に必要なのは「httpsで始まる公開URL」が1つできることです。

---

## 4. config.js を埋める

`config.js` を開き、次の3つを書き換えて保存し、公開先にもアップロードし直してください。

```js
GOOGLE_CLIENT_ID: "手順2でコピーしたクライアントID",
SPREADSHEET_ID: "手順1でメモしたスプレッドシートID",
RECEIPT_DRIVE_FOLDER_ID: "手順1でメモしたドライブフォルダID",
```

シート名や開始行が今後変わった場合も、このファイルの値を直すだけで対応できます。

---

## 5. 動作確認

1. スマホまたはパソコンで公開URLを開く。
2. 「Googleでログイン」→ 手順2でテストユーザーに登録したアカウントでログイン。
3. 仕訳入力タブで、テストの収入を1件入力 → スプレッドシートの一般会計シートに行が追加され、現金残高・口座残高が自動計算されることを確認。
4. 部費入金記録タブで、テストの部員×月を1件記録 → 部費入金管理シートのセルが更新されることを確認。
5. 上位大会タブで、特別会計シートの一覧が選べることを確認（大会用シートがまだなければ、まず元のExcelの手順で1つ作ってから試してください）。
6. スマホのブラウザで「ホーム画面に追加」を行うと、アイコンから全画面で起動できるようになります（PWA化）。

---

## 6. 引き継ぎ時にやること

新しい会計担当者が決まったら、次の3つを行ってください。アプリのコード自体は触りません。

1. スプレッドシートとドライブフォルダの共有設定に、新しい担当者のGoogleアカウントを追加する。
2. Google CloudのOAuth同意画面の「テストユーザー」に、新しい担当者のGoogleアカウントを追加する。
3. 新しい担当者に公開URLを伝える。

---

## 困ったときに

- **ログインできない**：OAuth同意画面の「テストユーザー」に、そのGoogleアカウントが登録されているか確認する。
- **「アクセスがブロックされました」と出る**：「承認済みのJavaScript生成元」に、実際にアプリを開いているURLが正確に登録されているか確認する（httpsとhttp、末尾のスラッシュの違いでも失敗します）。
- **入力してもスプレッドシートに反映されない**：`config.js` の `SPREADSHEET_ID` が正しいか、ログインしたアカウントにそのスプレッドシートの編集権限があるかを確認する。
- **ドロップダウンの選択肢が古い**：マスタシートの内容を直せば、次回読み込み時にアプリ側にも反映されます（アプリ側のコード修正は不要）。
